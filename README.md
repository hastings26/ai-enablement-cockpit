# The AI Cockpit
### A Hastings Hub prototype

The daily workspace for an AI Enablement leader. Decisions, programmes, dependencies, and signals — all on one calm surface.

**Live demo:** https://hastings26.github.io/ai-enablement-cockpit/

## What this is

An AI Enablement Director walks into a Monday with: a Board deadline driven by EU AI Act enforcement, a Copilot adoption number that isn't moving fast enough, a Champions network of 47 people across 12 business units that wants attention, six shadow-AI flags from the CISO, an ELT report due Wednesday, and a half-built agentic-workflows pilot that's been "in discovery" for three weeks.

The AI Cockpit is what would be on their second monitor.

## What's on the surface

- **Left rail — Programmes.** Every active programme by mode (Diagnose · Architect · Execute · Defend · Sharpen · Operating), with sponsor, next decision, and health.
- **Centre stage** —
  - **Hero**: the next *decision* that has to land. Not the next meeting. With a strategic frame in one line, the cost of inaction, and a one-click Cowork prompt that drafts the brief-up in SCR + Action Title format.
  - **Adoption strip**: the four numbers that actually matter — weekly active, prompts-per-user-per-day, time returned per active user, shadow-AI flags.
  - **This week**: the five things between now and Friday that move programmes through gates.
  - **Programme pipeline**: visual board with stuck-flags fired automatically when anything sits in pre-Execute mode >14 days with no shipped artefact.
  - **Outside the building**: regulation & benchmarks (EU AI Act, ISO 42001, Stanford, MIT) + vendor & model news (Anthropic, OpenAI, Microsoft). Every card carries a "why this matters" line that references *this* cockpit's programmes.
- **Right rail — Don't forget.** Overdue decisions, wobbling dependencies, planning-theatre flags, adoption signals, champion signals, benefits-realisation reviews. Six cards, ranked by leverage not by recency.

## How it would actually run

Single static HTML, deployable anywhere. In production:

1. A `/morning-briefing` skill runs at 06:00 each weekday against the user's tools (Copilot adoption telemetry, the CRM, the risk register, calendar, OneDrive). Output: a JSON file.
2. The cockpit reads the JSON on load — same surface, real data.
3. Every "Send to Cowork" button on the hero card drafts the actual brief-up using SCR + Action Title discipline (≤15 words per title, conclusion not topic, audience-specific framing).
4. Every "why this matters" news line is generated by cross-referencing the headline against the active programmes register — relevance ranked, not recency ranked.

No new platform. No new licence. The engine is whatever AI Cowork product the org already pays for; this is the cockpit that uses it well.
